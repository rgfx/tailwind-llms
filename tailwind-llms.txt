# Tailwind CSS v4.0 Documentation for LLMs

## Getting Started
Tailwind CSS v4.0 is a utility-first CSS framework that leverages modern CSS features like OKLCH colors, cascade layers, `@property`, and `color-mix()`. It uses a CSS-first configuration with the `@theme` directive, eliminating `tailwind.config.js`. Builds are up to 5x faster for full builds and 100x faster for incremental builds, measured in microseconds. All utilities are generated from theme variables, accessible as CSS custom properties (e.g., `var(--color-brand-500)`).

## Installation
- **Requirements**: Modern browsers (Safari 16.4+, Chrome 111+, Firefox 128+). Node.js and npm for CLI usage.
- **Steps**:
  1. Install via npm:
     ```bash
     npm install tailwindcss @tailwindcss/cli
     ```
  2. Create `input.css`:
     ```css
     @import "tailwindcss";
     @theme {
       --font-sans: "Inter", system-ui, sans-serif;
       --color-brand-500: oklch(0.637 0.237 25.331);
       --breakpoint-md: 48rem;
       --spacing: 0.25rem;
     }
     ```
  3. Build CSS:
     ```bash
     npx @tailwindcss/cli -i input.css -o output.css
     ```
  4. Include in HTML:
     ```html
     <link href="./output.css" rel="stylesheet">
     ```
- **Vite Plugin**:
  ```bash
  npm install @tailwindcss/vite
  ```
  Configure `vite.config.js`:
  ```javascript
  import tailwindcss from '@tailwindcss/vite'
  export default { plugins: [tailwindcss()] }
  ```

## Editor Setup
- **VS Code**: Install the Tailwind CSS IntelliSense extension for class autocompletion and hover previews.
- **Configuration**: Ensure `input.css` is referenced in your editor for accurate suggestions.
- **Custom Classes**: Add custom utilities/variants in `input.css` to ensure editor recognition:
  ```css
  @utility custom-space { margin-top: --value(--custom-space-*); }
  ```

## Compatibility
- **Browsers**: Safari 16.4+, Chrome 111+, Firefox 128+. v4.1 improves fallbacks for older browsers (e.g., Safari 15) for OKLCH colors and text shadows.
- **Node.js**: v16 or later recommended.
- **Frameworks**: Compatible with Vite, Next.js, Remix, and other modern frameworks via plugins.

## Upgrade Guide
- **From v3**: Use `npx @tailwindcss/upgrade` to migrate `tailwind.config.js` to CSS-first `@theme` configuration.
- **Key Changes**: No JavaScript config, automatic content detection, OKLCH colors, and new utilities (e.g., `text-shadow-*`, `mask-*` in v4.1).
- **Docs**: https://tailwindcss.com/docs/upgrade-guide

## Core Concepts
### Styling with Utility Classes
- Apply classes directly in HTML (e.g., `<div class="flex p-4 bg-brand-500">`).
- Utilities are generated from `--*` theme variables (e.g., `bg-brand-500` uses `var(--color-brand-500)`).

### Hover, Focus, and Other States
- Use state modifiers: `hover:`, `focus:`, `active:`, `disabled:`, etc.
- Example: `<button class="hover:bg-brand-700 focus:ring-2">`

### Responsive Design
- Use breakpoint prefixes (e.g., `sm:`, `md:`, `lg:`) from `--breakpoint-*`.
- Example: `<div class="text-base md:text-lg">`

### Dark Mode
- Enabled with `color-scheme: dark` or `[data-theme="dark"]`.
- Classes: `dark:{utility}` (e.g., `dark:bg-black`).
- Example:
  ```html
  <div class="bg-white dark:bg-black">
  ```

### Theme Variables
- Defined in `@theme`:
  ```css
  @theme {
    --color-lagoon: oklch(0.72 0.11 221.19);
    --font-body: "Roboto", sans-serif;
  }
  ```
- Accessible as `var(--color-lagoon)` and generate utilities (e.g., `text-lagoon`).

### Colors
- Uses OKLCH for vibrant, perceptually uniform colors (e.g., `oklch(0.637 0.237 25.331)`).
- Classes: `text-{color}`, `bg-{color}`, `border-{color}` (e.g., `text-brand-500`).

### Adding Custom Styles
- **Utilities**:
  ```css
  @utility tab-* { tab-size: --value(--tab-size-*); }
  @theme { --tab-size-4: 4; }
  ```
- **Variants**:
  ```css
  @custom-variant theme-midnight { &:where([data-theme="midnight"] *) { @slot; } }
  ```

### Detecting Classes in Source Files
- Automatic content detection scans template files (e.g., `.html`, `.js`, `.tsx`), ignoring `.gitignore` and binary files.
- Specify files:
  ```css
  @source "./src/**/*.{html,js,jsx,ts,tsx}";
  ```

### Functions and Directives
- **Directives**: `@theme`, `@source`, `@utility`, `@custom-variant`.
- **Functions**: `color-mix()`, `oklch()`, used internally for utilities.

## Base Styles
### Preflight
- Normalizes browser styles with `box-sizing: border-box`, removes margins, and sets font smoothing.
- Example:
  ```css
  @layer base {
    html { scroll-behavior: smooth; }
    body { font-family: var(--font-sans); background: var(--color-background); }
    img { max-width: 100%; height: auto; }
  }
  ```

## SVG
### Fill
- Classes: `fill-{color}` (e.g., `fill-brand-500` from `--color-*`).
- Example: `<svg class="fill-lagoon">`

### Stroke
- Classes: `stroke-{color}` (e.g., `stroke-brand-500` from `--color-*`).
- Example: `<svg class="stroke-lagoon">`

### Stroke Width
- Classes: `stroke-{size}` (e.g., `stroke-2` from `--stroke-width-*`).
- Example: `<svg class="stroke-1">`

## Accessibility
### Forced Color Adjust
- Classes: `forced-color-adjust-auto`, `forced-color-adjust-none`.
- Example: `<div class="forced-color-adjust-none">`

## Example Usage
```html
<!DOCTYPE html>
<html lang="en" class="scroll-smooth">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link href="./output.css" rel="stylesheet">
</head>
<body class="font-sans bg-brand-500 flex flex-col items-center p-4 dark:bg-black">
  <h1 class="text-3xl font-bold text-white text-shadow-md md:text-4xl hover:text-brand-700">Welcome</h1>
  <div class="grid grid-cols-1 md:grid-cols-3 gap-4">
    <div class="bg-white p-4 rounded-lg shadow-md">Item 1</div>
    <div class="bg-white p-4 rounded-lg col-span-2 filter blur-sm">Item 2</div>
  </div>
  <button class="px-4 py-2 border-2 border-white rounded-md transition-colors duration-300 hover:bg-white hover:text-black">Click Me</button>
  <svg class="fill-brand-500 stroke-white stroke-2 w-12 h-12" viewBox="0 0 24 24">
    <path d="M12 2L2 7l10 5 10-5-10-5z" />
  </svg>
</body>
</html>
```
